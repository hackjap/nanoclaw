<objective>
Add thread_ts awareness throughout the NanoClaw message pipeline so the bot replies in Slack threads instead of flattening threaded messages into channel-level responses.

Purpose: SLCK-01 and SLCK-02 require the bot to reply in threads. Currently slack.ts flattens all threaded replies (lines 82-85). This plan removes flattening, adds thread_ts to NewMessage, extends the Channel.sendMessage interface with optional options, and propagates thread_ts through IPC, router, container-runner, and index.ts wiring.

Output: Thread-aware message pipeline. Bot replies go to the correct thread. All existing tests updated. No new dependencies.
</objective>
